# 工作原理

pipguard 的核心原则：**扫描阶段绝不执行目标包代码**。

## 处理流程

1. 下载待安装包（wheel/sdist）
2. 解包并提取可分析文件
3. 基于 AST 做静态检测
4. 汇总风险并返回退出码
5. 若通过，再调用 pip 继续安装

## 检测重点

- 安装阶段可执行入口（如 `setup.py`）
- `.pth` 自动执行载荷
- 可疑系统调用、网络外连、动态代码执行
- 凭据窃取与环境探测相关模式

## 为什么更安全

- 不运行目标包代码，减少触发攻击面的机会
- 结果结构化，便于在 CI 中自动阻断
- 默认策略偏保守，优先保护供应链安全
